UC3 - Submit new Receipt
Data:
  • <ReceiptID>
  • <Date>
  • <Kind>
  • <Amount>
  • <Company>
  • <Country>
  • <City>
  • <Street>
  • <Number>
Main Flow:
  1. User issues `Add Receipt` command.
  2. System uses information provided to issue a new receipt.
  3. System saves receipt in memory.
  4. System executes UC5.
Existing ReceiptID (2)
  1. User is notified with: `Receipt ID already exists.`.
  2. Use case ends.
Invalid Date (2)
  1. User is notified with: `Please make sure your date is DD/MM/YYYY and try again.`.
  2. Use case ends.
Invalid Kind (2)
  1. User is notified with: `Please check receipt kinds and try again.`.
  2. Use case ends.
